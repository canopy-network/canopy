# CLIQ Operations

## CLIQ Transfer

Transfers liquid (already-vested) CLIQ between accounts.

```bash
canoliqctl cliq-transfer <from_address> <to_address_hex> <amount_uCLIQ>
```

Only vested CLIQ can be transferred. Vesting-locked CLIQ is not spendable.

## CLIQ Stake

Locks liquid CLIQ into a stake record, granting governance weight.

```bash
canoliqctl cliq-stake <address> <amount_uCLIQ>
```

**Effect**:
- Liquid CLIQ is debited
- `CLIQStake` record is created or aggregated
- `CanoliqGlobals.totalStakedCLIQ` is incremented
- `staked_at_height` is recorded for vote eligibility

:::tip Lock for more voting power
A stake can commit to a [vote-escrow lock tier](../tokenomics/vote-escrow.mdx) (`LOCK_3M`
through `LOCK_24M`) to multiply voting weight up to 4× and boost buyback rewards. Locks only
ever strengthen, and locked stake cannot be unstaked until its `lock_end_height`.
:::

## CLIQ Unstake

Queues an unstake, returning CLIQ to liquid balance after the unbond window.

```bash
canoliqctl cliq-unstake <address> <amount_uCLIQ>
```

**Effect**:
- `CLIQStake.amount` is decremented
- An `UnstakingCLIQ` record is queued with `mature_height`
- `CanoliqGlobals.totalStakedCLIQ` is decremented immediately
- Voting weight drops immediately (prevents flash-unstake attacks)

## CLIQ Claim Unstake

Claims a matured unstake, returning CLIQ to liquid balance.

```bash
canoliqctl cliq-claim-unstake <address> <unstake_id>
```

## CLIQ Claim Vested

Unlocks newly-vested CLIQ across all of the caller's vesting schedules.

```bash
canoliqctl cliq-claim-vested <address>
```

Reads the `VestingIndex` to find all schedules, then computes unlocked amounts for each.
